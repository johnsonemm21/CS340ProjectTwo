# CS340ProjectTwo
Global Rain Client/Server Application
In this project, the CRUD Python module was a key example of writing programs that are maintainable, readable, and adaptable. By separating database operations into a dedicated class, I was able to keep my dashboard code clean and focused only on user interaction and display logic. This made the code easier to read, debug, and update. One major advantage of this design is reusability—the same CRUD module could be used in other applications that need to interact with the same database without rewriting core logic. In the future, I could expand this module to include update and delete operations or reuse it in different projects such as web apps, APIs, or data analysis tools.

When approaching a problem, I focus on breaking the problem into smaller, manageable parts and understanding both the technical requirements and the user’s needs. For this project, I first analyzed Grazioso Salvare’s requirements, then designed the database queries and dashboard features to match those needs. This project differed from previous assignments because it required integrating multiple components using MongoDB, Python, and Dash rather than solving isolated problems. It pushed me to think more about system design and how different technologies work together. In the future, I would continue using strategies like planning data structures ahead of time, prototyping queries, and testing incrementally to ensure the database meets client expectations.
